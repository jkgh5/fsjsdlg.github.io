09. Review: Progress Report Fields

Reflections on Progress Reporter Fields

Reflections on Interpreting Progress Reporter Fields

Discussion on Progress Reporter

QUESTION:

Reflect

Take a closer look in the last exercise at the progress reporter that was generated using your Kafka server and Structured Streaming application. With the variations on applications, like maxRatePerPartition and trigger, how did the progress report change? What do they mean? Pick 3-5 fields, and write down what each field means.

ANSWER:

Thanks for your response. Here are my answers for what each field means:

  • id: ID of the current batch in UUID
  • currentDurationMs: cumulative times in milliseconds
  • currentTriggerStartOffsets: start offsets per source
  • sources.description: the description of source - kafka, rate, etc
  • sink: sink types (console, kafka, etc)
